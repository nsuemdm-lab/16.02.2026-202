### 1. Методическое пособие «Спринт-подготовка к защите за 120 минут»

**Цель:** Структурировать хаотичные знания в четкое пятиминутное выступление.

#### Блок 1. Бизнес-логика и актуальность (1 минута)
*   **Что сказать:** Не читайте название с экрана. Скажите, какую проблему решает ваше приложение. 
*   **Формула:** «Мой проект [Название] предназначен для [Целевая аудитория]. Он решает проблему [Проблема], позволяя автоматизировать [Процесс]».
*   **Акцент на «Фишку»:** Обязательно упомяните вашу индивидуальную задачу (например, фильтр совместимости или прогресс-бар).

#### Блок 2. Архитектура и структура данных (1 минута)
*   **Стек:** Четко назовите: PHP 8.x / Python 3.x, MySQL, Bootstrap 5.
*   **Хостинг:** Упомяните, что проект развернут на **Beget** (для PHP) или **PythonAnywhere** (для Python). Это подтверждает готовность к продакшену.
*   **БД:** Кратко опишите связи. «Сердце системы — таблица [Название], связанная с пользователями через `user_id` (связь один-ко-многим)».

#### Блок 3. Безопасность — ядро защиты (1.5 минуты)
Это самая важная часть для МДК.09.03. Вы должны подтвердить, что ваше приложение — не «решето».
*   **Input Validation:** «Все входящие данные проходят типизацию, например, через `(int)$_GET`».
*   **Anti-XSS:** «Вывод данных экранируется функцией `htmlspecialchars()`».
*   **Anti-SQLi:** «Используются подготовленные выражения (Prepared Statements) через PDO».
*   **Anti-CSRF:** «Формы защищены уникальными токенами в сессиях».
*   **Anti-IDOR:** «Доступ к заказам/профилям проверяется на уровне владения: `WHERE id = ? AND user_id = ?`».

#### Блок 4. Тестирование и качество кода (1 минута)
*   **SonarCloud:** Покажите скриншот или назовите рейтинг. «Код прошел аудит в SonarCloud, рейтинг надежности — A».
*   **Manual Testing:** Расскажите, как вы пытались ввести `' OR '1'='1` в поле логина и почему это не сработало.

#### Блок 5. Демонстрация (0.5 минуты)
*   Покажите ОДИН полный цикл: Регистрация -> Добавление сущности -> Отображение в ЛК.
